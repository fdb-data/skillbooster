## 协议与架构选型

### AG-UI（采纳：事件层）

- 引入 `@ag-ui/core` 的 **EventType 标准事件枚举**（RUN_STARTED / TEXT_MESSAGE_* / TOOL_CALL_* / CUSTOM / RUN_FINISHED / RUN_ERROR 等）
- 事件流跑在 Electron IPC 推送通道（`agent:event`，webContents.send）上，**不引入** CopilotKit 全家桶（本地单体应用无需远端解耦）
- CUSTOM 事件承载 SkillForge 扩展载荷：canvas_update / proposal / scene_draft
- 好处：不自造事件 schema，将来接远程 agent 或社区工具零成本

### A2UI（暂缓）

- Google A2UI v0.9 官方渲染器只有 Lit/Angular/Flutter，**React 渲染器未发布**（在官方 roadmap）
- 架构上预留：agent 工具协议可扩展 `render_ui`；待 React 渲染器发布后用于引导阶段动态表单
- 核心经验画布永远不走 A2UI——产品灵魂交互由自研 React Flow 画布承担

### React Flow（@xyflow/react）

- 七巧板画布的承载：自定义节点（知识积木/幽灵积木）、连线（流程顺序/关联）、元件栏拖入（HTML5 DnD + screenToFlowPosition）、小地图、缩放
- jsdom 测试需 mock ResizeObserver（tests/setup.ts）

### Markdown 渲染（自研，零依赖）

- 智能体回复的 Markdown 用自研轻量组件 `src/components/Markdown.tsx` 渲染（加粗/行内代码/分隔线/标题/无序列表/表格），**不引入** react-markdown 等完整依赖
- 理由：智能体输出的格式集合有限且可控，未识别语法按纯文本兜底即可；避免引入 remark 生态的体积与供应链面

## LLM 调用能力（electron/main/llm.ts）

| 能力 | 说明 |
|------|------|
| 流式输出 | SSE 解析，逐 delta 回调（onTextDelta），空闲超时按 chunk 重置 |
| Function Calling | OpenAI tools 格式；流式模式下聚合 tool_calls 增量 |
| 自动重试 | 5xx/超时/网络错误退避 2s/5s/10s；**429 限流单独用长退避 5s/21s/30s**（适配"每分钟 N 次"型限速窗口）；最多 3 次 |
| 速率限制 | 最小调用间隔 1.1s，不可绕过 |
| 中断 | 外部 AbortSignal，区分用户中断（ABORTED）与超时（TIMEOUT） |
| 降级兼容 | 模型不支持 tools 时（按 baseUrl+model 缓存探测），agentLoop 降级 JSON 协议 |

## TypeScript 配置

| 配置文件 | 目标 | 特点 |
|----------|------|------|
| tsconfig.node.json | 主进程 | ESNext, strict, 无 DOM |
| tsconfig.web.json | 渲染进程 | ESNext, strict, react-jsx, DOM, 路径别名 @/→src/ |
| tsconfig.json | 根 | 项目引用组合 node + web |

## 路径别名

- `@/` → `src/`（vite config 和 tsconfig.web.json 均已配置）

## 安全配置

- contextIsolation: true
- nodeIntegration: false
- 渲染进程通过 preload contextBridge 访问主进程能力
- CSP 策略限制 script/style/connect/img 来源

## 技术栈锁定原则

- 已选定的技术栈不更换，编译/运行问题排查根因解决
- better-sqlite3 为唯一数据库方案，不替换为 sql.js 等替代品
- LLM 调用内置速率限制（最小间隔 1.1s），遇 429/超时重试而非换方案
