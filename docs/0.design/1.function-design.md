# 功能设计

## 功能模块总览

| 模块 | 页面 | 核心功能 |
|------|------|----------|
| 场景管理 | Home | 创建/删除/进入场景 |
| 场景引导 | Guide | 引导智能体补全场景上下文（流式 + 属性实时同步） |
| 经验萃取 | Workbench | 智能体对话萃取 + 七巧板画布组装 + 参考文档 |
| A/B 验证 | Validate | 裸模型 vs 带 Skill 模型双流对比 + 分维度差异分析 |
| 导出部署 | Workbench 内 | 健康检查 + ZIP 打包 |
| 系统设置 | Settings | LLM 配置 + Agent 配置 |

---

## 一、场景管理（Home 页）

### 1.1 场景列表

- 展示所有场景卡片，3 列网格布局
- 每张卡片显示：场景名称、证据等级条（按条目数量和等级着色）、状态标签（active/completed）
- 按 updated_at 降序排列

### 1.2 创建场景

- 顶部输入框输入经验描述，Ctrl+Enter 或点击按钮开始
- 支持点击 📎 或拖拽上传附件（.txt/.md/.pdf/.docx）：附件以芯片展示（图标 + 文件名 + ✕ 删除），按路径去重
- 点击开始后：创建场景 → 附件逐个落库为**参考文档**（解析正文，引导/萃取智能体可读，进工作台后仍在参考文档面板）→ 进入引导页
- 只传文档不打字时，自动注入开场语"我上传了文档，请先读文档，然后引导我"

### 1.3 进入场景

- 点击卡片：有内容（canvas 非空）→ 进入 Workbench，无内容 → 进入 Guide

### 1.4 删除场景

- 卡片上的删除按钮，确认后删除场景及关联数据（canvas JSON、参考文档、对话记录）

## 二、场景引导（Guide 页）

### 2.1 引导对话

- 左侧对话区，AI 消息（流式打字，**轻量 Markdown 渲染**：加粗/分隔线/标题/列表/表格）+ 选项芯片 + 用户消息
- 引导智能体每轮先做**意图分析**：从用户首句和文档摘要里推断，能猜的不问
- 通过工具补全场景信息：
  - `update_scene`：更新场景草稿（name / protagonist / trigger / includes / excludes / projectName / done）
  - `ask_user`：提一个问题 + 2~4 个候选选项（最后固定「其他 · 直接说」），支持 `multiSelect` 多选，调用后本轮结束
- **选项交互**：
  - 单选：点击即发送
  - 多选（multiSelect=true）：芯片可勾选切换，「确认（N）」按钮把所选项用"、"拼接发送
  - 「其他 · 直接说」：点击展开**行内输入框**（自动聚焦，Enter/按钮提交）；多选模式下自填内容与所选项合并发送
- 场景草稿持久化（preferences 表），跨轮次/重启不丢失；返回引导页时通过 `guide:getDraft` 自动恢复
- 思考中状态：脉冲动画 + 实时工具状态（"正在更新场景属性…"）
- **错误兜底**：LLM 调用失败不静默——对话区显示友好错误气泡；429 限流特别提示"请等半分钟再发一次"

### 2.2 场景属性面板

- 右侧面板，实时显示和编辑：name / protagonist / trigger / includes / excludes
- 智能体调用 `update_scene` 工具时**实时同步**（CUSTOM scene_draft 事件），无需等本轮结束

### 2.3 引导完成

- 信息收集齐全后智能体置 done=true，输出总结
- 显示"进入萃取工作台"按钮，跳转 Workbench

## 三、经验萃取（Workbench 页）

### 3.1 布局与导航

- 顶栏导航：**场景定义 | 经验萃取（当前页）| 验证 | 导出/部署**——点「场景定义」返回引导页（草稿自动恢复）
- 左栏：参考文档面板（**可收起/展开**：「«」收起为 28px 竖条，「»」展开；概念树已随概念/关系类型移除）
- 中栏：七巧板经验画布（FlowCanvas，React Flow 驱动）
- 右栏：萃取对话（Conversation）

### 3.2 七巧板经验画布

- **知识积木节点**：三类知识块按分类着色（流程蓝 #3B82F6 / 规则深蓝 #2563EB / 洞察橙 #E08A2B），左侧色条 + 类型标签 + 证据等级圆点 + 待验证标签；旧数据中的概念/关系条目仍正常渲染（紫 #8B5CF6 / 绿 #10B981）
- **元件栏**（画布左上角）：流程/规则/洞察三种部件模板，拖入画布即建块并进入编辑态
- **行内编辑**：节点内直接改标题/内容/证据等级（四挡胶囊选择）
- **连线**：节点上下连接点拖拽连线；流程→流程自动识别为执行顺序（蓝色动画线），其余为关联（灰线）；选中连线按 Del/Backspace 删除
- **撤销/重做**：画布右上角 ↩↪ 按钮，快照栈上限 50，切换场景清栈
- **小地图**（按类型着色）+ 缩放控制 + 网格背景
- **自动排布**：无布局数据的条目（旧数据/agent 新增）按类型分列排布；用户拖动后位置持久化
- 新块高亮：`blockGrowIn` 飞入动画 + "刚长出"标签，3 秒淡出
- 顶部 Skill 进度指示（5 个圆点，每 3 块填 1 个）+ 块数统计

### 3.3 幽灵积木（提议）

- 智能体 `propose` 的内容渲染为**半透明虚线积木**，自动落在所属类型列已有块下方
- 三个操作：✓ 采纳（实体化 + 高亮，支持撤销）、✎ 改（节点内编辑后"改完采纳"）、✕ 不对（消散）
- 幽灵积木可拖动，采纳时在当前位置实体化
- 对话区显示提示："🧩 N 条提议已作为虚线积木放上画布"

### 3.4 萃取对话

- **开场引导卡**：对话为空时显示使用引导（三种方式：直接讲经验 / 文档起草 / 元件栏手搭）+「不知道从哪说起？让智能体先提问」按钮（一键触发智能体根据场景和缺口主动提问）
- 用户输入消息，萃取智能体通过**多步工具循环**响应：
  - `canvas_add`：添加用户明确确认的知识条目（直接上画布）
  - `canvas_update` / `canvas_delete`：修改/删除已有条目（按 id 定位）
  - `propose`：归纳推断的内容（幽灵积木待确认）
- **流式渲染**：回复文本逐字上屏（**轻量 Markdown 渲染**，含表格）；工具调用实时显示状态（"正在添加条目…"）
- **中断**：⏹ 按钮随时终止本轮
- **缺口驱动追问**：系统维护覆盖度摘要（三类条目分布 + 缺失类别 + 探索性条目清单）注入上下文，智能体优先围绕缺口提问
- **长对话压缩**：超过 24 条消息自动将较早历史摘要化（缓存于 preferences，按消息数失效）
- 自动滚动到底部

### 3.5 参考文档面板

- 上传文档（.txt/.md/.pdf/.docx，最大 20MB）；首页/引导页上传的附件也在此处
- 文档列表：勾选框（是否打入包）+ 文件名 + 删除按钮
- "从文档起草"按钮：智能体读文档后用 `canvas_add` 逐条建块（provenance 注明出处，证据等级按内容性质判断），生成开场白
- 面板可收起（「«」）为竖条（竖排"参考文档" + 文档数），点「»」展开，收起时画布获得更多宽度

## 四、A/B 验证（Validate 页）

### 4.1 对比布局

- 左右对比：A=裸模型输出，B=带 Skill 模型输出
- 底部输入测试指令

### 4.2 验证流程

1. 用户输入测试指令
2. 系统并行调用 LLM：裸模型（通用 system）和带 Skill 模型（system=SKILL.md 内容），**双流流式渲染**（逐字上屏 + 光标）
3. 双流结束后显示"正在逐维度分析差异…"
4. 第三次调用 LLM 做差异分析，输出**五维度结构化结果**

### 4.3 差异分析维度

| 维度 | 分析内容 |
|------|----------|
| 专业判断 | B 是否用上了 Skill 中的规则/洞察？引用了哪条？ |
| 操作性 | B 是否更可执行（具体步骤、阈值、检查点）？ |
| 边界意识 | B 是否更清楚什么情况不适用、何时升级求助？ |
| 风险提示 | B 是否提示了 A 没提到的风险信号？ |
| 幻觉与劣化 | B 有没有因 Skill 变差（生搬硬套、过度泛化）？ |

- 维度卡片网格展示 + 一句总评
- 实事求是原则：无差异就说无差异，B 更差直说
- 旧格式（整段 diffSummary）自动降级展示

### 4.4 保存验证记录

- "保存为验证记录"按钮，验证结果回流到萃取对话

## 五、导出部署（Workbench 内触发）

### 5.1 健康检查

- 检查项：
  - Canvas 是否为空（warning 级）
  - 参考文件是否缺失（error 级）
  - 对话是否为空（warning 级）
- 返回 passed/warnings 列表

### 5.2 ZIP 打包

- 弹出系统保存对话框
- ZIP 包内容：
  - `SKILL.md`：从 Canvas 生成的 Markdown（含 YAML frontmatter：name/description/version）
  - `experience-card.json`：Canvas JSON（含 layout 布局数据）
  - `references/`：所有 includeInPackage=true 的参考文档文件

## 六、系统设置（Settings 页）

### 6.1 LLM 配置

- Provider 列表管理：增删改 Provider（名称、baseUrl、apiKey、models）
- 连接测试：验证 API 可达性和密钥有效性
- 支持类型：OpenAI / Azure / 自定义端点（OpenAI 兼容）

### 6.2 智能体配置

- 三个 Agent 可独立配置：guide / extract / validate
- 每个 Agent 可配置：Provider、Model、API Key、Base URL；配置完整则覆盖全局配置（resolveAgentLLMConfig）
- 查看提示词文件（调用系统默认编辑器打开）
- 连接测试

### 6.3 通用设置

- 预留扩展位

## 数据模型

### ExperienceCard（经验卡片）

五类知识数组 + 可选布局（当前版本只启用 flows/rules/insights 三类；concepts/relations 字段保留、兼容旧数据，新增入口全部关闭）：

| 字段 | 类型 | 说明 |
|------|------|------|
| flows / rules / insights | KnowledgeEntry[] | 当前版本启用的三类知识条目 |
| concepts / relations | KnowledgeEntry[] | 企业级特性，当前版本暂不开放新增（字段保留） |
| layout | CanvasLayout（可选） | 画布布局：节点位置 + 连线。旧数据无此字段时自动排版 |

### KnowledgeEntry（知识条目）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | UUID |
| title | string | 是 | 条目标题（≤20字） |
| content | string | 是 | 条目内容 |
| verified | boolean | 是 | 是否已验证 |
| source | ai/user/doc | 否 | 来源标记 |
| evidenceLevel | institutional/validated/sample/exploratory | 否 | 证据等级 |
| provenance | string | 否 | 来源说明 |
| createdAt / updatedAt | string | 是 | 时间戳 |

### CanvasLayout（画布布局）

| 字段 | 说明 |
|------|------|
| positions | 条目 id → { x, y } 画布坐标 |
| edges | CanvasEdge[]：{ id, source, target, kind: 'flow-order' \| 'link', label? } |

### 证据等级

| 等级 | 含义 | 颜色 |
|------|------|------|
| institutional | 制度/规范级别 | #2E9E6B 绿色 |
| validated | 已验证的最佳实践 | #3B82F6 蓝色 |
| sample | 基于样本的经验 | #E0A93B 黄色 |
| exploratory | 探索性假设 | #E05D5D 红色 |

### CanvasOp（画布统一操作）

用户拖拽、行内编辑、提议采纳全部走这一套（智能体走主进程 canvasTools，二者最终都落到同一份 ExperienceCard）：

| 操作 | 参数 | 说明 |
|------|------|------|
| add | type, entry, position? | 添加条目（含画布位置） |
| update | id, patch | 修改条目字段 |
| delete | id | 删除条目（级联清理位置和连线） |
| move | id, position | 移动节点 |
| connect | edge | 建立连线（去重） |
| disconnect | edgeId | 删除连线 |

### CanvasUpdate（智能体 → 渲染端增量）

| 字段 | 说明 |
|------|------|
| type | 知识类型（智能体工具仅 flow/rule/insight；类型定义保留全部五类） |
| action | 操作（add/update/delete） |
| entry | KnowledgeEntry 完整对象 |

### Proposal（提议）

| 字段 | 说明 |
|------|------|
| id | UUID |
| type | 知识类型 |
| title / content | 提议内容 |

### 其余实体

Reference / ConversationMessage / Scene 结构不变（见 `src/contracts/ipc-types.ts`）。
