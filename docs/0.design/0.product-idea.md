# 产品构想

## 产品定位

SkillForge 是一款**隐性知识显性化**桌面工具。用户通过引导式对话描述自己的专业经验，智能体将其结构化为可视化的**经验画布**（七巧板式知识积木），并打包为可复用的 Skill 包。

## 核心问题

专业领域的隐性经验（"怎么做的"）长期停留在个人头脑中，难以传承、复用和验证。传统知识管理工具只管"存"，不管"萃"——存下来的文档没人看，真正有价值的判断逻辑和决策规则从未被结构化表达。

## 目标用户

- 各领域专家、资深从业者，拥有丰富但未结构化的实操经验
- 知识管理者，需要将团队经验转化为可复用资产
- AI 应用开发者，需要高质量 Skill 包增强大模型的专业能力

## 核心价值主张

1. **对话即萃取**：智能体通过多步工具调用直接操作画布——边对话边搭积木，用户全程可见
2. **七巧板画布**：经验以可拖拽的知识积木呈现，没有智能体也能纯手工拖拉拽组装
3. **三类知识积木**：流程（怎么做）、规则（怎么判）、洞察（怎么看）——聚焦个人经验萃取最有效的三类；概念/关系属企业级知识建模特性，当前版本暂缓（数据模型保留）
4. **诚实分界**：用户确认过的内容直接上画布；智能体归纳推断的内容以"幽灵积木"（半透明虚线）呈现，由用户原地采纳/拒绝/修改
5. **证据分级**：每条知识标注证据等级（制度级/已验证/样本级/探索性），区分可信度
6. **一键打包**：Skill 包（SKILL.md + experience-card.json + 参考文档）可直接部署给大模型使用
7. **A/B 验证**：裸模型 vs 带 Skill 模型双流对比，逐维度量化经验包的实际效果

## 产品形态

Electron 桌面应用，本地运行，数据存储在本地。不依赖云端服务，LLM 能力通过用户自行配置的 API 接入（OpenAI 兼容接口）。

## 核心工作流

```
输入经验描述 → 场景引导（2~4轮补全上下文）→ 经验萃取（智能体边对话边搭积木）
  → 画布组装（拖拽/连线/采纳提议）→ A/B 验证 → 打包导出
```

### 阶段一：场景引导

用户输入一段经验描述（可附文档，自动落库为参考文档）后，引导智能体先做**意图分析**（能推断的不问），通过 `update_scene` / `ask_user` 工具补全场景信息：谁在做、触发时机、边界（包含/排除），属性面板实时更新。选项支持多选；「其他 · 直接说」展开行内输入框。同时为项目起名。

### 阶段二：经验萃取

萃取智能体基于对话和参考文档，通过画布工具（`canvas_add` / `canvas_update` / `canvas_delete` / `propose`）增量构建 ExperienceCard。系统维护**覆盖度缺口摘要**驱动追问方向；确认内容直接上画布并高亮，归纳内容生成幽灵积木待确认。

### 阶段三：画布组装

用户在七巧板画布上自由组装：从元件栏拖入新积木、行内编辑、节点连线（流程顺序/概念关联）、采纳或拒绝幽灵积木。全部操作支持撤销/重做。

### 阶段四：A/B 验证

对同一测试指令，裸模型和带 Skill 模型**双流并行**生成回答（流式可见），随后逐维度分析差异：专业判断、操作性、边界意识、风险提示、幻觉与劣化。

### 阶段五：打包导出

健康检查通过后，一键导出 ZIP 包，包含 SKILL.md、experience-card.json（含画布布局）和参考文档。

## 与现有方案的差异

| 维度 | 传统知识库 | AI 对话工具 | SkillForge |
|------|-----------|------------|------------|
| 知识形态 | 文档/条目 | 对话记录 | 可视化知识积木（ExperienceCard） |
| 萃取方式 | 人工撰写 | 被动问答 | 智能体工具调用 + 手工拖拽组装 |
| 过程可见性 | 无 | 黑盒 | AG-UI 事件流：边说边搭积木 |
| 可信度标注 | 无 | 无 | 四级证据等级 + 提议/确认分界 |
| 可复用性 | 低 | 低 | Skill 包可直接部署 |
| 效果验证 | 无 | 无 | A/B 双流对比 + 五维度差异分析 |

## 发展方向

- 概念/关系知识类型：面向企业级知识建模（本体/图谱场景）时启用，数据模型已预留
- A2UI 接入：待官方 React 渲染器发布后，引导阶段支持智能体动态生成结构化表单
- 社区模板广场：共享和复用 Skill 包
- 多人协作：团队级经验萃取
- Skill 包市场：经验资产交易
- 更多 LLM 后端支持
