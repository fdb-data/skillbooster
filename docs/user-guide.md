## 1. 安装与首次启动

### 开发模式运行

```bash
# 环境要求：Node.js ≥ 18
npm install
npm run dev
```

### 打包安装程序

```bash
npm run package
# Windows：生成 SkillBooster-{version}-setup.exe
# macOS：生成 SkillBooster-{version}.dmg
```

数据存储位置：
- Windows：`%APPDATA%\SkillBooster\`
- macOS：`~/Library/Application Support/SkillBooster/`

## 2. 配置 LLM

**首次使用必须先完成 LLM 配置，否则智能体功能不可用。**

进入 **设置 → LLM 配置**：

1. 点击「添加 Provider」
2. 填写：
   - **名称**：自定义标识（如 OpenAI、DeepSeek）
   - **Base URL**：API 端点（如 `https://api.openai.com/v1`）
   - **API Key**：你的密钥
   - **模型**：输入模型名称（如 `gpt-4o`、`deepseek-chat`）
3. 点击「保存」

**支持所有 OpenAI 兼容接口**（OpenAI、Azure OpenAI、DeepSeek、Qwen、本地 Ollama 等）。

### 智能体配置（可选）

进入 **设置 → 智能体配置**，可为三个智能体分别指定不同模型：

| 智能体 | 用途 | 推荐 |
|--------|------|------|
| 引导 Agent | 场景信息补全，轻量对话 | 低成本快速模型 |
| 萃取 Agent | 核心萃取，需要工具调用能力 | 支持 function calling 的强模型 |
| 验证 Agent | A/B 对比分析 | 同萃取或更强 |

不单独配置则全部使用全局默认 Provider。

## 3. 工作流概览

```
① 首页：新建场景，输入经验描述（可附文档）
      ↓
② Guide 引导页：引导智能体补全上下文（2~4 轮）
      ↓
③ Workbench 工作台：
   左：参考文档面板
   中：七巧板经验画布（拖拽、连线、采纳提议）
   右：萃取对话（智能体边说边搭积木）
      ↓
④ Validate 验证页：裸模型 vs 带 Skill 双流对比
      ↓
⑤ 工作台顶栏「导出/部署」：健康检查 → 下载 ZIP
```

工作台顶栏可在四个阶段间随时切换，进度自动保存。

## 4. 场景管理（首页）

### 新建场景

1. 在顶部输入框里描述你的专业经验（一两句话即可）
2. 可选：点击 📎 或拖拽上传参考文档（`.txt` / `.md` / `.pdf` / `.docx`，上限 20MB）
3. 按 `Ctrl+Enter` 或点击「开始」

文档会自动解析并作为参考文档贯穿整个流程。只传文档不输入文字也可以，系统自动进入引导。

### 管理已有场景

- 点击场景卡片 → 有画布内容则进工作台，无内容则进引导页
- 卡片上的删除按钮 → 删除场景及所有关联数据（画布、对话、参考文档）

## 5. 场景引导（Guide）

引导智能体会分析你的描述，通过 2~4 轮对话补全：

- **主体**：谁在做这件事
- **触发时机**：什么情况下发生
- **包含/排除**：边界范围

### 选项交互

- **单选选项**：点击即发送
- **多选选项**：勾选多个后点「确认（N）」
- **「其他 · 直接说」**：点击展开文本框，可自由输入

### 引导完成

智能体收集完毕后输出总结，点击「进入萃取工作台」进入下一阶段。

## 6. 经验萃取（Workbench）

### 布局说明

| 区域 | 说明 |
|------|------|
| 左侧 | 参考文档面板（可点 `«` 收起） |
| 中间 | 七巧板经验画布 |
| 右侧 | 萃取对话区 |

### 六种知识积木

| 类型 | 颜色 | 含义 |
|------|------|------|
| 流程 | 蓝 | 怎么做 · 步骤序列 |
| 规则 | 深蓝 | 怎么判 · 决策规则 |
| 洞察 | 橙 | 怎么看 · 信号与模式 |

### 发起萃取

在右侧对话框输入问题或描述，智能体会：
1. 流式输出分析与提问
2. 通过工具调用直接在画布上新增/修改/删除积木
3. 对归纳推断的内容生成**幽灵积木**（半透明虚线）待你确认

### 画布操作

- **拖入新积木**：从左下方元件栏拖入画布
- **行内编辑**：双击积木直接改内容
- **连线**：鼠标悬停节点边缘出现连接点，拖拽连线
- **采纳幽灵积木**：点击幽灵积木上的「✓ 采纳」按钮，实体化上画布
- **撤销 / 重做**：`Ctrl+Z` / `Ctrl+Shift+Z`（或工具栏按钮）
- **小地图**：右下角缩略图，可拖拽定位

### 参考文档

- 左侧面板上传文档后，萃取智能体可检索文档内容
- 「从文档起草」：智能体直接读文档，批量生成初始积木
- 勾选文档可决定是否打包进导出的 Skill 包

### 中断

对话进行中点击「中断」按钮，智能体停止当前轮次；已操作的画布内容保留。

## 7. A/B 验证（Validate）

点击工作台顶栏「验证」进入。

1. 输入一个测试问题（模拟真实使用场景）
2. 点击「开始验证」
3. 左右双列**同时流式输出**：
   - 左：**裸模型**（不带 Skill）
   - 右：**带 Skill 模型**

验证完成后，系统从 5 个维度分析差异：

| 维度 | 说明 |
|------|------|
| 专业判断 | 是否体现领域专业知识 |
| 操作性 | 回答是否可执行 |
| 边界意识 | 是否知道自己能做什么、不能做什么 |
| 风险提示 | 是否提示潜在风险 |
| 幻觉与劣化 | 带 Skill 后是否引入了错误信息 |

## 8. 打包导出

点击工作台顶栏「导出/部署」。

1. **健康检查**：系统检查画布内容完整性
2. **选择参考文档**：在左侧参考文档面板勾选需要打包的文档
3. 点击「下载 Skill 包」，保存 ZIP 文件

---

## 9. Skill 包格式说明

导出的 ZIP 包含：

```
{场景名}-skill.zip
├── SKILL.md                 # YAML frontmatter + 结构化知识 Markdown
│                            # 可直接作为大模型 system prompt 使用
├── experience-card.json     # 完整结构化经验卡（含证据等级、来源、画布布局）
└── references/              # 你勾选打包的参考文档原件
```

**SKILL.md 使用方式**：将其内容粘贴到大模型对话的 system prompt，或通过 API 注入 `messages[0].role=system`，即可让大模型以你的经验视角回答问题。

## 10. 常见问题

**Q：智能体没有响应 / 报错**
- 检查「设置 → LLM 配置」是否填写了正确的 Base URL 和 API Key
- 确认模型名称拼写正确
- 如报 429（限流），稍等 30 秒后重试，系统会自动退避重试

**Q：LLM 不支持工具调用（function calling）**
- SkillBooster 会自动检测并切换为 JSON 降级协议，功能基本一致，但响应略慢

**Q：画布操作没有保存**
- 画布数据 300ms 防抖自动落盘，正常关闭窗口前会自动保存

**Q：场景引导阶段想重新开始**
- 点击工作台顶栏「场景定义」回到引导页，草稿会自动恢复；发新消息即可重新引导

**Q：如何在大模型中使用导出的 Skill 包**
- 把 `SKILL.md` 的内容作为 system prompt 注入即可
- 若使用 Dify、FastGPT、Coze 等平台，直接粘贴到「系统提示词」区域
